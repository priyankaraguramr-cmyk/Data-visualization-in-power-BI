# Data-visualization-in-power-BI
Power BI Assignment 2 – DAX & Data Visualization
Overview
This assignment demonstrates the implementation of Power BI data modeling, DAX calculations, and interactive dashboard development using sales and target datasets. The objective was to transform raw business data into meaningful insights through calculated columns, measures, relationships, and visual reports.
The solution includes data preparation, DAX-based calculations, KPI tracking, category-level analysis, geographic analysis, and sales performance monitoring.
Project Objectives
The primary objectives of this assignment are:
•	Import and prepare sales datasets in Power BI.
•	Build a proper data model with relationships.
•	Create calculated columns using DAX.
•	Develop business measures using DAX.
•	Design interactive dashboards for business analysis.
•	Analyze sales performance against targets.
•	Visualize geographic and category-wise sales distribution.
•	Generate business insights through KPIs and visualizations.
Dataset Information
The assignment utilizes the following datasets:
Orders Data
Contains transactional sales information including:
•	Order ID
•	Order Date
•	Customer Name
•	Category
•	City
•	State
•	Amount
•	Profit
•	Quantity
•	Profit Margin
Order Details
Contains detailed order-level information used for advanced calculations and reporting.
Sales Target
Contains category-wise target values used to compare actual sales performance against planned targets.
Total Target by Month
Contains monthly target information used for performance tracking over time.
Data Modeling
A data model was created by establishing relationships between the imported tables.
Relationships Created
•	Orders Data ↔ Order Details
•	Orders Data ↔ Sales Target
•	Orders Data ↔ Total Target by Month
•	Date Table ↔ Orders Data
The relationship model ensures accurate aggregation and filtering across all visualizations.
DAX Calculations
Calculated Column 1: Revenue per Order
Purpose:
Calculate the revenue generated for each order.
Formula:
Revenue per Order =
'Order Details'[Amount] * 'Order Details'[Quantity]
Calculated Column 2: Sales Category

Purpose:
Classify sales performance into Above Average and Below Average categories.
Business Benefit:
Provides quick identification of high-performing and low-performing sales transactions.
DAX Measures
The following measures were created to support reporting and KPI analysis.
Total Amount
Calculates total sales amount.
Total Profit
Calculates overall profit generated.
Total Quantity
Calculates total quantity sold.
Order Count
Calculates total number of orders.
Max Profit
Identifies the highest profit generated by a sub-category.
Average Profit in Delhi
Calculates average profit for transactions occurring in Delhi.
Target by Month
Used for monthly sales target comparisons.
Dashboard Development
Three report pages were designed.
Page 1 – Core Dashboard
Purpose:
Provide an executive-level overview of sales performance.
Visualizations
1.	Sales Target Achievement by Category
•	Compares actual sales against target values.
2.	Monthly Sales Trend
•	Displays sales performance over time.
3.	Max Profit by Sub-Category
•	Highlights the most profitable sub-categories.
4.	Profit vs Quantity by Sub-Category
•	Analyzes the relationship between quantity sold and profit generated.
5.	Total Sales Amount KPI
•	Displays overall sales value.
6.	Total Sales Target KPI
•	Displays overall sales target.
Business Insight
This dashboard provides a quick summary of organizational sales performance and target achievement.
Page 2 – Advanced Analysis
Purpose:
Perform deeper sales target and performance analysis.
Visualizations
1.	Minimum Target by Segment
•	Identifies segments with the lowest target values.
2.	Sales Performance Matrix
•	Compares sales and targets across categories and months.
Business Insight
Helps management evaluate category-wise performance and identify areas requiring improvement.
Page 3 – Geographic & Distribution Analysis
Purpose:
Analyze sales distribution geographically and by product categories.
Visualizations
1.	Geographic Sales Analysis
•	Displays sales distribution across locations.
2.	Sales Distribution by Sub-Category
•	Shows category contribution to overall sales.
3.	Order Count by State
•	Compares order volume across different states.
Business Insight
Provides visibility into regional sales performance and customer demand patterns.
Key Findings
•	Sales performance varies across categories and sub-categories.
•	Certain product groups contribute significantly higher profits.
•	Geographic distribution reveals concentration of sales in specific regions.
•	Monthly target comparison helps identify performance gaps.
•	KPI monitoring provides quick visibility into business performance.
